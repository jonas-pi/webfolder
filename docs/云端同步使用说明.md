# 云端同步使用说明

本应用通过 **GitHub Gist** 或 **Gitee 代码片段** 存储与 [本衍生版浏览器扩展 BookmarkHub](https://github.com/jonas-pi/BookmarkHub) 相同格式的书签数据（`SyncDataInfo`）。下面说明如何在两端创建云端片段，以及本应用选项里的**字段名称**（与界面文案一致）。

---

## 一、本应用里要填什么（字段对照）

路径：**我的 → 云端同步**。

| 界面上的名称 | 含义 | 注意 |
|-------------|------|------|
| **同步源** | 选 **GitHub Gist** 或 **Gitee 片段** | 与扩展里的「云端类型」对应；两种云端**片段 ID、令牌不可混用**。 |
| **访问令牌** | GitHub：**Personal Access Token**；Gitee：**私人令牌** | 仅保存在本机，请勿泄露。 |
| **片段 ID** | 云端为该片段分配的唯一标识 | 只填 **ID 字符串**，不要填整段浏览器 URL。 |
| **片段内文件名** | 该片段里**那一个 JSON 文件**的文件名 | 多设备、多客户端须**完全一致**；默认建议 **`BookmarkHub`**（无扩展名也可，与创建片段时填写的文件名一致即可）。 |

若同时使用 **浏览器扩展**（衍生版 BookmarkHub），请与扩展选项中的设置对齐：

| 本应用（鸿蒙） | 浏览器扩展（常见命名） |
|----------------|------------------------|
| 同步源：GitHub / Gitee | `gistProvider`：`github` / `gitee` |
| 访问令牌 | `githubToken`（扩展里字段名未改，两种云端都用它存令牌） |
| 片段 ID | `gistID` |
| 片段内文件名 | `gistFileName` |

## 二、GitHub：创建 Gist 与令牌

### 1. 创建 Personal Access Token（访问令牌）

1. 浏览器登录 GitHub，打开 **Settings**（用户设置）。
2. 左侧进入 **Developer settings** → **Personal access tokens**。
3. 使用 **Tokens (classic)**（经典令牌）时：点 **Generate new token (classic)**，勾选 **`gist`** 权限（说明为对 Gist 的完整操作权限），生成后复制令牌（常以 `ghp_` 开头）。  
   - 若使用 **Fine-grained token**，请在令牌权限中为 **Gists** 勾选读写能力（以 GitHub 当前页面选项为准）。
4. 将令牌粘贴到本应用 **访问令牌** 输入框。

### 2. 新建 Secret Gist（私密 Gist）

1. 打开 **[https://gist.github.com/](https://gist.github.com/)**。
2. 在编辑区 **Filename including extension…** 中填写与 **片段内文件名** 一致的名称，例如 **`BookmarkHub`**（可不写 `.json`，但与各端配置保持一致即可）。
3. 文件内容可先填入 **`{}`** 占位，或留空后由本应用首次 **上传到云端** 写入完整数据（若创建页要求非空，用 `{}` 即可）。
4. 选择 **Create secret gist**（**私密**，避免书签数据公开）。
5. 创建成功后，浏览器地址栏 URL 形如：  
   `https://gist.github.com/<用户名>/<gist_id>`  
   其中 **最后一段** `<gist_id>`（一串十六进制字符）即为本应用中的 **片段 ID**。

### 3. 本应用中的设置小结（GitHub）

- **同步源**：**GitHub Gist**  
- **访问令牌**：上述 PAT  
- **片段 ID**：URL 最后一段  
- **片段内文件名**：与 Gist 里该 JSON 文件名一致（默认 **`BookmarkHub`**）

---

## 三、Gitee：创建代码片段与私人令牌

Gitee 将对应能力称为 **代码片段**（Open API 路径仍为 **`/gists/...`**，与 GitHub 概念对应）。

### 1. 创建私人令牌（访问令牌）

1. 登录 Gitee，进入 **设置** → **安全设置** → **私人令牌**（或直接访问 [https://gitee.com/personal_access_tokens](https://gitee.com/personal_access_tokens)）。
2. 点击生成新令牌，在权限（scope）中勾选与 **代码片段 / gists** 相关的项（Open API 文档中常见 scope 为 **`gists`**；若界面为中文分组，请勾选能 **读写自己的代码片段** 的权限，**以 Gitee 当前页面为准**）。
3. 生成后复制令牌，粘贴到本应用 **访问令牌** 输入框。

### 2. 新建私密代码片段

1. 在 Gitee 顶部或 **+** 菜单中找到 **新建代码片段**（或进入 **代码片段** 再新建，具体入口以当前网站为准）。
2. **可见性** 请选择 **私密**，避免书签数据被公开。
3. 添加一个文件，**文件名**与 **片段内文件名** 一致，例如 **`BookmarkHub`**。
4. 内容可先填 **`{}`**，保存后在本应用执行 **上传到云端** 写入正式书签结构。
5. 保存后打开该片段页面，地址栏 URL 中会包含 **片段 ID**（通常为路径中的一段标识符，**不是**完整 `https://…`）。只把 **ID 字符串** 填到本应用的 **片段 ID**。

### 3. 本应用中的设置小结（Gitee）

- **同步源**：**Gitee 片段**  
- **访问令牌**：Gitee **私人令牌**  
- **片段 ID**：该代码片段的 ID（仅 ID，非整段 URL）  
- **片段内文件名**：与片段里 JSON 文件名一致（默认 **`BookmarkHub`**）

---

## 四、使用注意（准确理解行为）

1. **数据格式相同，凭据与 ID 不通用**：GitHub Gist 与 Gitee 代码片段里存的 JSON 结构一致，但 **GitHub 的 Token / 片段 ID 不能用于 Gitee**，反之亦然；切换 **同步源** 时必须整套重配。  
2. **片段内文件名** 必须与 API 更新时使用的文件名一致；若 Gist/片段里没有同名文件，首次 **上传** 时通常会由接口创建或更新该文件（以各平台 API 行为为准），但多端请务必统一名称。  
3. **上传到云端** 会覆盖远端该文件名对应的内容；**从云端拉取** 会覆盖本机书签缓存。首次使用前建议先 **导出/备份** 本地书签（如有 HTML 导出或其它备份方式）。  
4. 本应用通过 **HTTPS** 调用 `api.github.com` 或 `gitee.com/api/v5`，令牌仅用于应用内请求，请妥善保管设备安全。

---

## 五、接口与术语（供核对）

| 平台 | 认证方式（本应用实现） |
|------|------------------------|
| GitHub | `Authorization: Bearer <token>`，REST **Gists** API。 |
| Gitee | 查询参数 **`access_token=<token>`**，OpenAPI v5 **`/gists/{id}`** 等。 |

界面中的 **「片段」** 在 GitHub 侧即 **Gist**，在 Gitee 侧即 **代码片段**；**片段 ID** 均指平台为单个 Gist/代码片段分配的唯一标识符。
