# 云端同步使用说明

本应用通过 **GitHub Gist**、**Gitee 代码片段** 或 **WebDAV 文件** 存储与 [本衍生版浏览器扩展 BookmarkHub](https://github.com/jonas-pi/BookmarkHub) 相同格式的书签数据（`SyncDataInfo`）。下面说明如何创建云端位置，以及本应用选项里的**字段名称**（与界面文案一致）。

---

## 一、本应用里要填什么（字段对照）

路径：**我的 → 云端同步**。

| 界面上的名称 | 含义 | 注意 |
|-------------|------|------|
| **同步源** | 选 **GitHub Gist**、**Gitee 片段** 或 **WebDAV** | 三种云端的凭据与定位信息不可混用。 |
| **访问令牌** | GitHub：**Personal Access Token**；Gitee：**私人令牌** | 仅保存在本机，请勿泄露。 |
| **片段 ID** | 云端为该片段分配的唯一标识 | 只填 **ID 字符串**，不要填整段浏览器 URL。 |
| **片段内文件名** | 该片段里**那一个 JSON 文件**的文件名 | 多设备、多客户端须**完全一致**；默认建议 **`BookmarkHub`**（无扩展名也可，与创建片段时填写的文件名一致即可）。 |
| **WebDAV 文件地址** | 指向 BookmarkHub JSON 文件的完整 URL | 例如 `https://dav.example.com/BookmarkHub/bookmarks.json`；这里填**文件 URL**，不要只填目录。 |
| **WebDAV 用户名 / 密码** | WebDAV 登录凭据 | 建议使用服务商提供的应用密码；仅保存在本机。 |

若同时使用 **浏览器扩展**（衍生版 BookmarkHub）的 Gist/Gitee 同步，请与扩展选项中的设置对齐：

| 本应用（鸿蒙） | 浏览器扩展（常见命名） |
|----------------|------------------------|
| 同步源：GitHub / Gitee | `gistProvider`：`github` / `gitee` |
| 访问令牌 | `githubToken`（扩展里字段名未改，两种云端都用它存令牌） |
| 片段 ID | `gistID` |
| 片段内文件名 | `gistFileName` |

## 二、GitHub：创建 Gist 与令牌

### 1. 创建 Personal Access Token（访问令牌）

1. 浏览器登录 GitHub，打开 **Settings**（用户设置）。
2. 左侧进入 **Developer settings** → **Personal access tokens**。
3. 使用 **Tokens (classic)**（经典令牌）时：点 **Generate new token (classic)**，勾选 **`gist`** 权限（说明为对 Gist 的完整操作权限），生成后复制令牌（常以 `ghp_` 开头）。  
   - 若使用 **Fine-grained token**，请在令牌权限中为 **Gists** 勾选读写能力（以 GitHub 当前页面选项为准）。
4. 将令牌粘贴到本应用 **访问令牌** 输入框。

### 2. 新建 Secret Gist（私密 Gist）

1. 打开 **[https://gist.github.com/](https://gist.github.com/)**。
2. 在编辑区 **Filename including extension…** 中填写与 **片段内文件名** 一致的名称，例如 **`BookmarkHub`**（可不写 `.json`，但与各端配置保持一致即可）。
3. 文件内容可先填入 **`{}`** 占位，或留空后由本应用首次 **上传到云端** 写入完整数据（若创建页要求非空，用 `{}` 即可）。
4. 选择 **Create secret gist**（**私密**，避免书签数据公开）。
5. 创建成功后，浏览器地址栏 URL 形如：  
   `https://gist.github.com/<用户名>/<gist_id>`  
   其中 **最后一段** `<gist_id>`（一串十六进制字符）即为本应用中的 **片段 ID**。

### 3. 本应用中的设置小结（GitHub）

- **同步源**：**GitHub Gist**  
- **访问令牌**：上述 PAT  
- **片段 ID**：URL 最后一段  
- **片段内文件名**：与 Gist 里该 JSON 文件名一致（默认 **`BookmarkHub`**）

## 三、Gitee：自动配置代码片段与私人令牌

Gitee 将对应能力称为 **代码片段**（Open API 路径仍为 **`/gists/...`**，与 GitHub 概念对应）。

### 1. 创建私人令牌（访问令牌）

1. 登录 Gitee，进入 **设置** → **安全设置** → **私人令牌**（或直接访问 [https://gitee.com/profile/personal_access_tokens](https://gitee.com/profile/personal_access_tokens)）。
2. 点击生成新令牌，在权限（scope）中勾选与 **代码片段 / gists** 相关的项（Open API 文档中常见 scope 为 **`gists`**；若界面为中文分组，请勾选能 **读写自己的代码片段** 的权限，**以 Gitee 当前页面为准**）。
3. 生成后复制令牌，粘贴到本应用 **Gitee 私人令牌** 输入框。

### 2. 自动配置私密代码片段

1. 选择 **同步源 → Gitee 片段**。
2. 填写 **Gitee 私人令牌**。
3. 点击 **用当前令牌自动配置**。
4. 应用会优先复用包含默认文件名（**`BookmarkHub`**）的已有代码片段；若不存在，则自动创建私密代码片段并保存 **片段 ID**。
5. 如需接入已有片段，可展开 **高级手动配置**，手动填写 **片段 ID** 与 **片段内文件名**。

### 3. 本应用中的设置小结（Gitee）

- **同步源**：**Gitee 片段**  
- **访问令牌**：Gitee **私人令牌**  
- **片段 ID**：自动配置会写入；手动配置时填该代码片段的 ID（仅 ID，非整段 URL）  
- **片段内文件名**：默认 **`BookmarkHub`**，需与其它端一致

## 四、WebDAV：文件地址与应用密码

1. 在你的网盘或 WebDAV 服务里准备一个 JSON 文件，内容可先填 **`{}`**。
2. 把该文件的完整 WebDAV 地址填入 **WebDAV 文件地址**；这里需要文件 URL，不是目录 URL。
3. 用户名与密码使用 WebDAV 凭据；如果服务商支持应用密码，优先使用应用密码。
4. **上传到云端** 会通过 `PUT` 覆盖该文件；**从云端拉取** 会通过 `GET` 读取该文件。

### 本应用中的设置小结（WebDAV）

- **同步源**：**WebDAV**
- **WebDAV 文件地址**：远端 JSON 文件完整 URL
- **用户名**：WebDAV 用户名
- **密码或应用密码**：WebDAV 密码或应用密码

---

## 五、使用注意（准确理解行为）

1. **数据格式相同，凭据与定位信息不通用**：GitHub Gist、Gitee 代码片段与 WebDAV 文件里存的 JSON 结构一致，但 **GitHub 的 Token / 片段 ID 不能用于 Gitee 或 WebDAV**，反之亦然；切换 **同步源** 时必须整套重配。
2. **片段内文件名** 必须与 API 更新时使用的文件名一致；若 Gist/片段里没有同名文件，首次 **上传** 时通常会由接口创建或更新该文件（以各平台 API 行为为准），但多端请务必统一名称。  
3. **上传到云端** 会覆盖远端该文件名对应的内容；**从云端拉取** 会覆盖本机书签缓存。首次使用前建议先 **导出/备份** 本地书签（如有 HTML 导出或其它备份方式）。  
4. 本应用通过 **HTTPS** 调用 `api.github.com`、`gitee.com/api/v5` 或你填写的 WebDAV 文件地址，令牌/密码仅用于应用内请求，请妥善保管设备安全。

## 六、接口与术语（供核对）

| 平台 | 认证方式（本应用实现） |
|------|------------------------|
| GitHub | `Authorization: Bearer <token>`，REST **Gists** API。 |
| Gitee | 查询参数 **`access_token=<token>`**，OpenAPI v5 **`/gists/{id}`** 等。 |
| WebDAV | `Authorization: Basic <base64(username:password)>`，对目标文件执行 `GET` / `PUT`。 |

界面中的 **「片段」** 在 GitHub 侧即 **Gist**，在 Gitee 侧即 **代码片段**；**片段 ID** 均指平台为单个 Gist/代码片段分配的唯一标识符。WebDAV 侧直接对应一个远端文件。
